Project Name: Autonomous Air Hockey Robot

Project Description:

The autonomous air hockey robot is a project dedicated to improving the training experience of air hockey hobbyists and enthusiasts. This project will involve a system composed of a PC, a microcontroller, a camera sensor, two laser sensors, two motors, six pulleys, and an LCD. The motors will be attached to cables via a CoreXY system. The cables will also be attached to a paddle, so that the paddle can be moved to block and attack a puck. The camera sensor will be placed above the air hockey table so that it looks down on the playing field resembling a bird's-eye view. The camera sensor will detect and track a hockey puck in motion, providing the pucks coordinates to the PC. The PC will use this data and an algorithm to predict the trajectory of the moving puck. Following this, the PC will communicate with the microcontroller to share data on the pucks position and trajectory. The microcontroller will determine where the paddle is in relation to the puck, and will command the motors to move the paddle in a defensive position. If a puck is at any point scored, the laser sensors will detect the score and transmit the data to the microcontroller. The microcontroller will then communicate with the LCD so that the LCD display reflects the correct score for the user.

Stretch Project Specific Design Requirements (SPSDRs):

  1. SPSDR #1 (Hardware): An ability to produce sound using speakers in the scoreboard system..
  2. SPSDR #2 (Hardware): An ability to change the speed of the robot via button.

Project Specific Design Requirements (PSDRs):

  1. PSDR #1 (Hardware): An ability to use a microcontroller to control the XY directions of a gantry system using PWM and 2 motors.
  2. PSDR #2 (Hardware): An ability to interface with an IC to establish communication and send data between a microcontroller and a PC via UART.
  3. PSDR #3 (Hardware): An ability to interface a seven-segment display scoreboard with shift registers.
  4. PSDR #4 (Hardware): An ability to keep score with a microcontroller by counting the number of times an infrared laser beam is broken.
  5. PSDR #5 (Software): An ability to estimate the position and trajectory of a puck in motion using computer vision.
