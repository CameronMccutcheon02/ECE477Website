The first problem I wanted to address is demonstrating the IR sensor. I looked up an IR sensor circuit to see what was already out there and found this layout. Source: https://circuitdigest.com/electronic-circuits/ir-sensor-circuit-diagram

Test circuit for infrared Led display. Potentiometer is used to control the range of the IR proximity sensor. This pivots the design away from a break sensor to a proximity sensor. This will be easier to keep on one PCB and be simpler to work with. I used an AD2 to test this circuit and get some voltage measurements to see how the sensor works.

Resting state for the IR Sensor. Blue output voltage doesnt have enough voltage to power the LED. (~2.5V is off, ~4.5 is on). Yellow voltage is what is passed into the op amp.

Close proximity to IR sensor increases the yellow voltage passed through to the op amp. The blue output voltage is high enough to power the output LED.

Started by looking for already existing designs for a laser tripwire system and found this DIY design for a tripwire alarm system. This looked promising as something I could adapt. I could redesign this for lower power consumption and interface with a microcontroller by replacing the speaker output with the microcontroller. There are a few problems with using a 555 timer like in the image. While placing a delay on how long it takes to reset with laser tripwire seems like a good idea, its better to place that functionality into the microcontroller because it is much easier there. Using a 555 timer in this manner also requires a manual reset that isnt practical for a multipoint scoring game.

Designed this layout for the scoring detection system. The input voltage into the microcontroller will drop when the light is blocked, and the microcontroller can count that as a goal. The score can be maintained as a constant output in the output signal. PCB can be designed to contain the microcontroller and the phototransistor with the laser contained across the gap. Values can be changed later on, but this is a good start. Initial microcontroller Im using use for testing is the STM32 that we used in 362.
